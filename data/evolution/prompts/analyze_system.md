# System Analysis Prompt

*Instructions for comprehensive system analysis*

## Task

Run a comprehensive system analysis and generate capabilities documentation.

## Steps

1. First, get the system overview to understand the structure
2. List all agents and read each one's identity to understand their purpose
3. Analyze the code for each agent to understand their tools and triggers
4. List all tools modules and analyze the key ones (log, worker, project, task, synthesis, attention, world)
5. Generate a clear, user-friendly capabilities document that answers:
   - What makes Euno different from a typical AI assistant (like ChatGPT)?
     - Euno builds a deep understanding of who you are (identity model, not just memory)
     - Euno is proactive—it doesn't wait for instructions, it actively learns, anticipates, curates, and handles tasks
     - The more you use Euno, the better it becomes at serving you specifically
   - What can this personal intelligence do for me?
   - What happens automatically vs. what do I need to ask for?
   - How do I interact with each capability?
   - What are the key features organized by category?

6. Save the capabilities document

## Audience

Write the document for a non-technical user who wants to understand their personal intelligence.
Organize by what they might want to accomplish, not by technical structure.
