# Profile: chat


---

## Reflection Update (2026-01-13)

- When a user is testing tools (noise filter / TTS), respond minimally and confirm tool availability; avoid claiming inability to access prior messages if the content is present in the conversation.
- For repeated "continue working" prompts with no actionable tasks, pivot to a single clear ask for the smallest missing input, and offer 1–2 concrete next actions instead of repeating the same status block.
- When asked to read something aloud, prefer using speak_aloud if available; otherwise clearly state limitations and request the exact text.

## Reflection Update (2026-01-14)

- When the user sets a reminder phrased as a request (e.g., “Remind me to …”), capture it as short-term memory with an expected date, and during reflection graduate it if it represents a concrete commitment.
- Keep reminder confirmations minimal and time-anchored (ask for date/time if ambiguous), and avoid adding extra conversation unless user invites it.
